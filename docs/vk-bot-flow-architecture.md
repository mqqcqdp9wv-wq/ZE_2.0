# VK-бот ZE.Studio — Архитектура потоков

## Три типа пользователей и что они видят

### 1. Новый пользователь (никогда не писал в сообщество)

```
Заходит в сообщения группы ВКонтакте
        ↓
Видит кнопку «Начать» (стандартная VK-механика)
        ↓
Нажимает → бот получает событие message_new с payload: {"command":"start"}
        ↓
Бот показывает welcome-экран с inline-меню
        ↓
Бот отправляет второе сообщение с фиксированной клавиатурой «☰ Меню»
```

**Кнопка «Начать» настраивается:** Управление сообществом → Сообщения → Приветствие

---

### 2. Старый пользователь (уже писал, кнопки «Начать» больше нет)

```
Открывает сообщения группы
        ↓
Видит внизу чата фиксированную кнопку «☰ Меню»
        ↓
Нажимает → бот получает message_new с текстом «☰ Меню»
        ↓
Бот распознаёт триггер → показывает welcome-экран
```

**Фиксированная клавиатура** устанавливается ботом при первом открытии welcome
и остаётся видна внизу чата навсегда — независимо от прокрутки переписки.

---

### 3. Пользователь написал текст напрямую

```
Написал в чат: «Сколько стоит тонировка Камри?»
        ↓
Бот получает событие message_new
        ↓
Не является триггером, нет payload
        ↓
Бот → пользователю: «Получили ваш вопрос! Елена ответит в ближайшее время 🙂»
Бот → Елене в Telegram: уведомление с текстом + ссылкой на профиль ВК
```

**Важно:** бот НЕ возвращает меню в ответ на свободный текст.
Человек не хочет идти через бота — не надо его заставлять.

---

## Общая схема решений

```
Входящее событие от пользователя
        │
        ├── payload с target (нажал inline-кнопку бота)
        │       └── Навигация к нужному узлу меню
        │
        ├── Текст = триггер («привет», «меню», «начать», «☰ меню»)
        │       └── Показать welcome-экран + установить fixed keyboard
        │
        └── Свободный текст (всё остальное)
                ├── Пользователю: «Получили ваш вопрос, Елена ответит»
                └── Елене в Telegram: текст сообщения + ссылка на профиль
```

---

## Два типа клавиатур в VK

| Тип | Как выглядит | Когда исчезает |
|-----|-------------|---------------|
| **Inline** | Кнопки прикреплены к конкретному сообщению | При прокрутке уходит вверх |
| **Фиксированная** | Полоса кнопок внизу чата, над полем ввода | Только если бот явно уберёт |

**Используем оба:**
- Inline — для навигации внутри меню (все экраны бота)
- Фиксированная — кнопка «☰ Меню» как постоянный якорь для возврата

---

## Список триггеров → welcome

Бот открывает welcome-экран если пользователь написал любое из:

- `начать`
- `start`
- `привет`
- `меню`
- `в начало`
- `начало`
- `☰ меню` (нажатие фиксированной кнопки)

---

## Настройка кнопки «Начать» в VK

1. Управление сообществом → **Сообщения**
2. Включить сообщения, если не включены
3. Раздел **«Кнопка начала диалога»** → задать название (например «Начать» или «Открыть меню»)
4. При нажатии VK отправляет `payload: {"command":"start"}` — бот его ловит

## Уведомления Елене — формат

```
💬 [ВК] Сообщение клиента

👤 [ссылка на профиль ВК]

📝 Сколько стоит тонировка Камри?
```

Елена видит уведомление в Telegram, заходит в VK, отвечает вручную.

## Env-переменные для работы уведомлений

```
VK_TOKEN_TEST=       токен тестовой группы
VK_CONFIRMATION_TEST= строка подтверждения Callback URL
TELEGRAM_TOKEN=      токен Telegram-бота для уведомлений
ADMIN_ID=            Telegram chat_id Елены
```
